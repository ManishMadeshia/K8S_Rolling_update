Rolling Update - Version 2

In Version 2, Kubernetes upgrades your application by gradually replacing old Pods (v1) with new Pods (v2). This ensures zero downtime.

Basic Commands

# Check current deployment
kubectl get deployments

# Update deployment to new image (v2)
kubectl set image deployment/my-app my-container=nginx:1.16.1

# Monitor rollout status
kubectl rollout status deployment/my-app

# Undo (rollback) if needed
kubectl rollout undo deployment/my-app
